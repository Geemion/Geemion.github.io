Windows核心编程读书笔记2（第4章 进程）

文章目录

##主要内容：
什么是进程、系统如何创建进程内核对象、如何管理进程、如何使用内核对象对进程进行操作、进程不同属性、如何结束运行的进程

##什么是进程

  • 进程是一个正在运行的程序实例，进程由两个部分组成：进程内核对象、进程地址空间(执行文件代码和栈堆等动态空间)
  • 进程内核对象存放关于进程的统计信息，进程地址空间存放执行模块/Dll模块代码和数据（还包含动态分配的空间线程堆栈和堆分配的空间）
  • 进程是不活泼，由线程完成操作运行（线程负责执行包含在进程的地址空间中的代码）。每个线程都有它自己的一组CPU寄存器和它自己的堆栈。每个进程至少
    拥有一个线程，来执行进程的地址空间中的代码。如果没有线程来执行进程的地址空间中的代码，那么进程就没有存在的理由了，系统就将自动撤消该进程和它的地址空间。

##Windows应用程序

  • /SUBSYSTEM:CONSOLE GUI　/SUBSYSTEM:WINDOWS　　CLI
  • 应用程序入口点
  • 加载到进程地址空间的每个可执行文件或D L L文件均被赋予一个独一无二的实例句柄。HINSTANCE和HMODULE是完全相同的对象（只是在16位系统中不同）,表示映像加载到内存中的基址 可通过Getmodulehandle获得

##CreateProcess函数

  • 当一个线程调用CreateProcess时，系统就会创建一个进程内核对象，其初始使用计数是1,进程内核对象不是进程本身，而是操作系统管理进程时使用的一个较小的数据结构。进程内核对象可以看作是由进程统计信息组成的一个较小的数据结构。然后创建一个虚拟地址空间、创建一个线程内核对象。
  • LPTSTR lpCommandLine CreateProcess期望你将传递一个非常量字符串的地址(2000以上版本可以随意)。 /GF 传递参数时一般为”WORDPAD README>TXT”

  • 当路径非全路径时，搜索可执行文件路径顺序

  • lpProcessAttributes 进程内核对象安全属性 lpThreadAttributes 线程内核对象安全属性，一般可以直接传NULL

  • dwCreationFlags 参数用于标识标志，以便用于规定如何来创建新进程。如果将标志逐位用O R操作符组合起来的话，就可以设定多个标志。
  • 父进程创建子进程时使用的lpStartupInfo，在子进程中可以使用GetStartupInfo来查询

  • lpEnvironment 参数用于指向包含新进程将要使用的环境字符串的内存块。在大多数情况下，为该参数传递NULL，使子进程能够继承它的父进程正在使用的一组环境字符串

  • lpProcessInformation 参数用于指向你必须指定的PROCESS_INFORMATION结构。CreateProcess在返回之前要对该结构的成员进行初始化，创建新进程可使系统建立一个进程内核对象和一个线程内核对象。在创建进程的时候，系统为每个对象赋予一个初始使用计数值1。必须关闭子进程和它的主线程的句柄，以避免在应用程序运行时泄漏资源。当然，当进程终止运行时，系统会自动消除这些泄漏现象，但是，当进程不再需要访问子进程和它的线程时，编写得较好的软件能够显式关闭这些句柄（通过调用Closehandle函数来关闭）。
    由于某些原因，许多开发人员认为，关闭进程或线程的句柄，会促使系统撤消该进程或线程。实际情况并非如此。关闭句柄只是告诉系统，你对进程或线程的统计数据不感兴趣。进程或线程将继续运行，直到它自己终止运行。

##终止进程运行

  • 主线程的进入点函数返回（最好使用这个方法）。

  • 进程中的一个线程调用ExitProcess函数（应该避免使用这种方法）。

  • 另一个进程中的线程调用TerminateProcess函数（应该避免使用这种方法）。

  • 进程中的所有线程自行终止运行（这种情况几乎从未发生）。

  • GetExitCodeProcess
  • 子进程 pi.hTread pi.hProcess WaitFoxSingleObject(pi.hProcess,INFINITE),如果想建立独立子进程关闭两个句柄。进程和线程结束后，句柄对象被标记为激活， WaitForSingleObject会返回
    ##管理员以标准用户权限运行时
  • 从vista以后，管理员登录系统后会建立两个令牌（Token），高权限的和标准用户。进程默认启动权限为标准用户
  • 管理员用户Run as提权时系统会在一个安全桌面显示确认对话框 等待用户确认 系统进程蓝色 签名的灰色 未签名橙色
  • 普通用户权限提升时要求输入管理员账户和口令
  • 带盾牌图样的应用程序点击后直接需要权限提升，必须生成一个新的进程实例才能权限提升。但是一个未提升权限进程可以创建一个权限提升的进程在通过进程间通信IPC调用
  • 自动提升权限及不用run as(通过嵌入清单文件、兼容性设置) 手动提升run as/ShellExecuteEx
  • 提升权限后的进程通过createprocess创建的进程仍然是权限提升的
  • GetProcessElevation P112 IsUserAnAdmin 获得Token—–>pElevationType—->CheckTokenMemberShip
  • GetProcessTime查看进程时间，GetProcessIoCounters查看IO次数
  • GetProcessImageFileName返回内核格式的文件名

    、AccessChk icacls.exe /setintegritylevel
